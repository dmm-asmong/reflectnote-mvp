모바일 헤더 / 네비게이션 방식

현재 헤더가 모바일에서 화면의 50% 이상을 차지합니다. 어떻게 바꿀까요?

A

하단 탭 바 (Bottom Tab Bar)

모바일에서는 상단 헤더를 앱 이름 + 로고 수준으로 줄이고, 네비게이션을 화면 하단으로 이동. 네이티브 앱처럼 엄지로 조작하기 편함.

  • 네이티브 앱 느낌

단점

  • 데스크톱과 레이아웃 구조 크게 달라짐
  • 개발량 증가
B

헤더 접기 (Compact Header)

모바일에서 헤더 문구·설명을 숨기고, 앱 이름 + 네비게이션 4개만 간결하게 유지. 현재 구조를 최대한 유지하면서 공간 확보.

  • 네비게이션 항상 보임

단점

  • 헤더가 여전히 공간 차지
  • 하단 탭보다 덜 직관적
C

헤더 유지 + 폼/콘텐츠만 최적화

헤더는 현재대로 두고, 각 페이지의 폼·카드·버튼 레이아웃만 모바일 친화적으로 조정. 헤더 스크롤 후 콘텐츠 집중.

  • 네비게이션 접근성 낮음
